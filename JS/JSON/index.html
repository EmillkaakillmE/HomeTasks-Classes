Введение

JSON - JavaScript Object Notation - Описание объекта в JavaScript. Был создан для того, чтобы без труда передавать данные с сервера в виде строки и превращать их в JS объекты на клиентской стороне

Данные в этом формате содержатся парами "Ключ: Значение". Между ключом и значение обязательно ставится знак ":". Пары ключ-значение отделяются друг от друга запятыми.

Запись/объект - это неупорядоченное множество пар ключ-значение, заключенное в фигурные скобки "{}".
Массив - это упорядоченное множество значенией. Массив заключается в квадратные скобки "[]".

Массивы можно записывать в объектах

Синтаксис JSON очень строгий и все ключи должны быть указаны именно в двойных ковычках. Численые и булевые значения без ковычек вообще. Все строковые значения должны быть также только в двойных ковычках. Массивы, тут как обычно, в квадратных скобках.

У JS объектов такого строго синтаксиса нет. Объекты и их ключи могут быть записаны в любых кавычках, у ключей этих ковычек может вообще не быть. Соответственно сразу парсить такой объект через JSON не получится. Но можно превратить этот JS объект в валидный для JSON вид, с помощью функции Stringify()

Также можно вычёркивать те или иные параметры из объекто перед тем, как превращать их в JSON формат. Передавать в stringify первым параметр сам объект, чтобы превратить его в JSON. А потом через запятую следующими параметрами передать ключи тех элементов, которые мы хотим видеть в нашем JSON. Передавать такие параметры нужно в квадратных скобках, как будто это массив этих параметров (по сути это и есть массив), и каждый параметр должен быть обёрнут в двойные ковычки.

Ещё один параметр в stringify просто преобразует JSON объект в красивом виде с отступами. Именно количество отступов мы и указываем третьим параметром

В JSON нет смысла передавать методы, символьные свойства и ключи со значением undefined.
